## Optimization Rationale

Identified a low-risk read-path bottleneck: repeated inode attribute queries on stat/getattr/fstat-heavy paths after lookup/readdir_plus. Implemented a conservative shared LRU inode attribute cache with explicit invalidation on create, write, truncate, chmod, chown, utimens, unlink, rename, mkdir, rmdir, symlink, and link. Overlay partial-origin direct inode metadata updates now invalidate the delta attr cache too.

Added profiling counters:

- `attr_cache_hits`
- `attr_cache_misses`
